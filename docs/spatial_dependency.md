# Spatial Dependency Experiment — Reduced-Spatial-Identity Model

Final robustness experiment before feature lock (per explicit instruction:
single comparison, no architecture changes, no additional variants).
Generated by `backend/app/models/spatial_dependency.py` against the real
validation split.

## Setup

- **Model A** — the existing Phase 3 winning CatBoost model, loaded from
  `ml/models/classifier_catboost.cbm`. **Not retrained.**
- **Model B** — exactly ONE new training run, identical in every way to
  Model A except `h3_cell` and `geohash` are removed from the categorical
  feature set. Everything else kept: hotspot density (`hotspot_frequency`,
  `violation_density`, `junction_density`, `police_station_density`),
  rolling activity (`violations_last_15/30/60m`, `same_hour_previous_day`,
  `rolling_hotspot_intensity`), temporal features (`hour`, `weekday`,
  `hour_sin/cos`, `is_peak_hour`), aggregated historical statistics
  (`junction_historical_risk`, `offence_historical_risk`,
  `vehicle_type_historical_risk`, `center_code_historical_risk`), and the
  organizational categoricals (`junction_name`, `police_station`,
  `center_code`, `vehicle_type`, `primary_offence_code`, `primary_violation_type`)
  — these are NOT direct spatial-grid IDs and were deliberately kept.

## Results (validation split)

| | PR-AUC | Precision | Recall | F1 |
|---|---|---|---|---|
| **Model A** (full feature set, with `h3_cell`) | 0.8767 | 0.7316 | 0.9620 | 0.8311 |
| **Model B** (`h3_cell`/`geohash` removed) | 0.8719 | 0.7330 | 0.9644 | 0.8329 |
| **Drop** | **0.55%** | — | — | — |

## Verdict: **Spatial abstraction = PASS** (drop ≤ 3%)

## Honest interpretation — this does NOT contradict the Phase 3.5 spatial holdout FAIL

At first glance this looks inconsistent with `docs/spatial_holdout.md`
(7.88% PR-AUC drop on unseen H3 cells, FAIL). It isn't — the two
experiments measure different things:

- **Spatial holdout (Phase 3.5 Task 3)** asks: *"if a cell was never seen
  during training at all, how much does performance degrade?"* On a
  genuinely new cell, `hotspot_frequency`, `violation_density`,
  `junction_historical_risk`, etc. all start from zero/cold-start values too
  — there's no history for ANY per-cell feature, identity-based or not. That
  experiment's failure reflects a **cold-start problem affecting nearly
  every spatial feature**, not something unique to `h3_cell`'s raw identity.

- **This experiment (Reduced-Spatial-Identity)** asks: *"for cells the
  model already has data on, how much does the raw grid-ID categorical
  column itself add, on top of every other feature that already describes
  that cell's behavior?"* The answer is: almost nothing (0.55%). The
  information `h3_cell` was carrying is **largely redundant** with the
  density/rolling/historical-risk features computed per-cell and the
  organizational categoricals (`junction_name`, `police_station`) that are
  already strong location proxies.

**Combined reading:** the model isn't fragile because it memorizes
`h3_cell` specifically — it's fragile because most of its strongest
features (identity-based or not) are historical aggregates that don't exist
yet for a brand-new location. Removing `h3_cell` doesn't fix that
underlying cold-start sensitivity (spatial holdout would likely still show
a similar drop with Model B's feature set, untested here per the "no
additional variants" instruction) — it just shows that `h3_cell` itself
isn't doing irreplaceable work for ALREADY-SEEN locations.

## Decision

**Feature set is now FROZEN per instruction.** `h3_cell` is kept (this
experiment, run specifically to check whether it could be safely dropped,
found doing so buys negligible robustness benefit while giving up a small
but real amount of accuracy — not a good trade). The spatial-holdout
limitation documented in `docs/spatial_holdout.md` stands as a known,
disclosed limitation; per instruction this result does **not** block
deployment. If geographic coverage genuinely expands in the future,
retraining on data from the new area remains the correct fix — not a
feature-set change.
